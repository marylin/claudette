# Claudette v1.1 — Remaining Work Plan

## Summary
Most v1.1 backend work is already complete (claude bridge, sessions, git, usage, agents, settings all real). Three gaps remain: terminal PTY (completely missing), file/git watchers (no auto-refresh), and project persistence (no config-manager). This plan covers only the remaining work.

## Audit of v1.1 Features Plan vs Current State

| Phase | Status | Notes |
|---|---|---|
| 1 — Claude CLI Bridge | **DONE** | Real spawn, stdout parsing, permission detection, process lifecycle |
| 2 — Session Manager | **DONE** | Real JSONL parsing, project listing, session delete |
| 3 — Project Management | **TODO** | No config-manager.ts, no recent projects, no restore-last-project |
| 4 — File System | **PARTIAL** | readDir/readFile real, but NO chokidar watcher |

| 9 — Integration | **BLOCKED** | Blocked on phases 3, 4, 6 |

## Tasks

### Phase A — Project Persistence (was Phase 3)
1. [S] Create `src/main/config-manager.ts` — stores recent projects + last-open to `~/.claudette/config.json`
2. [S] Wire `project:get-current`, `project:get-recent` IPC handlers — returns data from config-manager
3. [S] Restore last project on app launch in `index.ts` — reads config, emits `project:changed`
4. [M] Implement project-change lifecycle — all managers reinitialize when project path changes
5. [S] Add preload API methods for project IPC channels — `getCurrentProject`, `getRecentProjects`

### Phase B — File System Watcher (was Phase 4 gap)
6. [S] Add `chokidar` dependency — `npm install chokidar`
7. [M] Create `src/main/fs-watcher.ts` — watches project dir, emits `fs:tree-updated` to renderer, debounced 200ms
8. [S] Wire watcher lifecycle to project changes — start/stop watcher on project switch
9. [S] Add preload listener for `fs:tree-updated` — renderer gets notified when files change

### Phase C — Git Auto-Refresh (was Phase 5 gap)
10. [S] Add git status auto-refresh — reuse fs-watcher events to debounce-emit `git:status-updated`
11. [S] Add preload listener for `git:status-updated` — renderer can subscribe

### Phase D — Terminal PTY (was Phase 6)
12. [M] Add `node-pty` dependency + electron-rebuild — native module for Electron
13. [M] Create `src/main/pty-manager.ts` — spawns PowerShell/bash PTY, forwards data via IPC
14. [M] Wire terminal IPC: `terminal:start`, `terminal:input`, `terminal:resize`, `terminal:kill`
15. [S] Add preload API for terminal IPC — `startTerminal`, `sendTerminalInput`, `resizeTerminal`, `onTerminalData`
16. [M] Update `TerminalPanel.tsx` — connect xterm.js to real PTY via IPC (keystrokes→main→PTY, PTY→main→xterm)
17. [S] Wire PTY lifecycle to project changes — kill + respawn PTY in new project dir

### Phase E — CLAUDE.md Enhancements (was Phase 8 gap)
18. [S] Add global CLAUDE.md scope to `claude-md:read`/`write` — support `~/.claude/CLAUDE.md`
19. [S] Watch CLAUDE.md for external changes — notify renderer to reload if file changes on disk

### Phase F — Smoke Test
20. [M] Walk through all 10 PRD success criteria — fix any bugs found

## Dependencies
- Phase B before Phase C (git auto-refresh reuses file watcher)
- Phase D is independent — can be done in parallel with A/B/C
- Phase F after all others

## Open Questions
1. **node-pty on Windows**: Requires Visual C++ Build Tools for native compilation. Should we use `electron-rebuild` postinstall or bundle pre-built binaries? (Recommendation: `electron-rebuild` — standard approach)
2. **chokidar version**: v3 (stable, well-tested) or v4 (ESM-only, may have Electron CJS issues)? (Recommendation: chokidar@3 for CJS compatibility)
